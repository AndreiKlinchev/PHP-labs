# Лабораторная работа №6

## Тема: Обработка и валидация форм в PHP

## Цель работы

Изучить работу HTML-форм в PHP, освоить отправку данных на сервер, обработку `POST`-запросов, валидацию данных и сохранение информации в файл.

## Выбранная тема проекта

Для лабораторной работы была выбрана тема: **каталог странных фактов**.

Пользователь может:
- добавить новый факт через HTML-форму;
- проверить корректность введённых данных;
- сохранить запись в файл;
- просмотреть все сохранённые записи в виде таблицы;
- отсортировать записи по выбранному полю.

## Модель данных

В проекте используется следующая модель данных:

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | название факта |
| `category` | enum | категория факта |
| `fact_date` | date | дата, связанная с фактом |
| `author` | string | имя автора или того, кто добавил факт |
| `source` | string | источник информации |
| `description` | text | подробное описание факта |
| `rating` | number | оценка странности от 1 до 10 |
| `is_true` | checkbox | отметка, похож ли факт на правдивый |
| `created_at` | date/time | дата и время добавления записи |

### Почему эта модель подходит под условие

- есть больше 6 полей;
- есть поля типа `string`;
- есть поле типа `date`;
- есть поле с ограниченным набором значений `category`;
- есть поле типа `text` для длинного описания.

## Структура проекта

Проект состоит из следующих файлов:

- [index.php](./index.php) — главная страница с HTML-формой;

- [style.css](./style.css) — оформление страниц.

## Описание реализации

### 1. HTML-форма

На странице `index.php` создана форма для добавления нового факта.  
Форма отправляет данные методом `POST` в файл `save.php`.

В форме используются следующие элементы:
- текстовые поля;
- выпадающий список `select`;
- поле даты;
- поле `textarea`;
- числовое поле;
- `checkbox`.

### 2. Клиентская валидация

Для улучшения удобства пользователя в форме использованы стандартные HTML-атрибуты:

- `required` — обязательные поля;
- `minlength` — минимальная длина строки;
- `maxlength` — максимальная длина строки;
- `min` и `max` — диапазон для числового значения.

Такая проверка помогает сразу сообщить пользователю об ошибке ещё до отправки формы.

### 3. Серверная обработка данных

После отправки формы данные попадают в `save.php`.  
В этом файле:

1. проверяется, что запрос пришёл методом `POST`;
2. создаются объекты классов валидации и сохранения;
3. вызывается метод сохранения;
4. при ошибках пользователь возвращается на форму и видит сообщения;
5. при успехе показывается сообщение об успешном сохранении.

Для сохранения данных использована сессия:
- старые значения формы сохраняются, чтобы не вводить их заново;
- ошибки сохраняются и выводятся после перенаправления;
- сообщение об успехе тоже передаётся через сессию.

### 4. Валидация на сервере

Проверка данных вынесена в класс `FormValidator`.

Проверяются:
- заполненность обязательных полей;
- минимальная длина строк;
- корректность категории;
- правильный формат даты;
- диапазон числа от 1 до 10.

Это нужно, потому что клиентскую проверку можно обойти, а серверная валидация делает приложение более надёжным.

### 5. Сохранение данных

Данные сохраняются в файл `data/facts.json` в формате JSON.

Пример одной записи:

```json
{
  "title": "Осьминог имеет три сердца",
  "category": "animals",
  "fact_date": "2024-05-10",
  "author": "Ivan",
  "source": "Internet",
  "description": "У осьминога действительно три сердца и голубая кровь.",
  "rating": 8,
  "is_true": 1,
  "created_at": "2026-04-10 12:30:00"
}
```

JSON был выбран потому, что:
- он читаемый;
- его удобно использовать в PHP;
- его легко расширять в следующих лабораторных работах.

### 6. Вывод данных

Для просмотра сохранённых записей создан файл `list.php`.

На этой странице:
- данные считываются из JSON-файла;
- каждая запись выводится в HTML-таблицу;
- доступны ссылки для сортировки по нескольким полям.

Сортировка реализована через параметр `sort` в адресной строке.  
Например:

```text
list.php?sort=title
list.php?sort=created_at
```

Для сортировки используется функция `usort()`.

## ООП-реализация

Для получения более высокой оценки решение было сделано с использованием объектно-ориентированного подхода.

### Использованные классы

#### `FormValidator`

Отвечает за проверку входных данных формы.

Основные задачи:
- принять массив данных;
- проверить значения;
- вернуть список ошибок.

#### `FactRepository`

Отвечает за чтение и запись данных в JSON-файл.

Основные задачи:
- получить все записи;
- добавить новую запись;
- создать папку для хранения данных при необходимости.

#### `FactManager`

Связывает валидацию и сохранение.

Основные задачи:
- получить данные формы;
- вызвать валидатор;
- при отсутствии ошибок сформировать запись;
- передать запись в репозиторий.

### Почему был выбран ООП-подход

ООП делает код более понятным и удобным для расширения:
- валидация хранится отдельно от логики сохранения;
- работа с файлом вынесена в отдельный класс;
- код легче читать и изменять;
- в будущем можно добавлять новые функции без сильной переделки проекта.

## Объяснение решения простыми словами

Смысл работы приложения такой:

1. Пользователь открывает страницу с формой.
2. Вводит данные о странном факте.
3. После нажатия кнопки данные отправляются в `save.php`.
4. PHP проверяет, правильно ли заполнены поля.
5. Если есть ошибка, пользователь получает сообщение и исправляет форму.
6. Если всё правильно, запись сохраняется в файл `facts.json`.
7. На странице `list.php` можно увидеть все сохранённые записи.

Таким образом, проект показывает полный цикл работы с формами:
- ввод;
- отправка;
- получение на сервере;
- проверка;
- сохранение;
- отображение.

## Контрольные вопросы

### 1. Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?

Основные методы отправки данных на сервер — это `GET` и `POST`.

- `GET` передаёт данные через адресную строку;
- `POST` передаёт данные в теле запроса.

HTML-форма стандартно поддерживает методы:
- `GET`;
- `POST`.

### 2. Какие глобальные переменные используются для доступа к данным формы в PHP?

Для доступа к данным формы в PHP используются суперглобальные массивы:

- `$_GET` — для данных, отправленных методом `GET`;
- `$_POST` — для данных, отправленных методом `POST`;
- `$_REQUEST` — содержит объединённые данные из `$_GET`, `$_POST` и `$_COOKIE`.

На практике чаще всего лучше использовать именно `$_GET` или `$_POST`, чтобы точно понимать источник данных.

### 3. Как обеспечить безопасность при обработке данных из формы?

Для защиты данных формы нужно:

- проверять и валидировать входные данные на сервере;
- экранировать вывод с помощью `htmlspecialchars()`, чтобы защититься от XSS;
- не доверять только клиентской валидации;
- проверять типы и диапазоны значений;
- по возможности использовать защиту от CSRF в более серьёзных проектах;
- не выполнять пользовательский ввод как код или SQL-запрос без фильтрации.

В данном проекте защита от XSS реализована через функцию `htmlspecialchars()` при выводе данных на страницу.
